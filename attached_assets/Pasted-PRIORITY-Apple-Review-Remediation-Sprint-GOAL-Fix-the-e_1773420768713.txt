KNOWN ROOT CAUSES (audit should confirm these)
==================================================

BUG 1 — SIGN IN WITH APPLE
Error seen by Apple reviewer: "Firebase: The audience in ID Token [com.marvelcardvault.app] does not match the expected audience com.marvelcardvault.web."
Root cause: Firebase is configured with a Web Client ID but the Apple Sign-In token issued for the iOS bundle ID (com.marvelcardvault.app). The nonce or audience parameter being passed to Firebase does not match the iOS client configuration.

BUG 2 — IAP PURCHASE ERROR
Error seen: "Purchase Issue: Subscription product not found. Please try again later."
Known product ID in App Store Connect: MCV_Apple_Superhero
The product must be loaded and confirmed available before the purchase button is enabled. The error fires on dashboard load — meaning product initialization is failing silently before the user even taps purchase.

BUG 3 — UGC REPORT MECHANISM
A "Report" option must be visible on all user-generated content surfaces the Apple reviewer can access (marketplace listings, listing detail pages, user profiles tied to listings). Previous implementation may exist — audit before rebuilding.

================================================
PHASE 2 — FIX SIGN IN WITH APPLE
==================================================

The Firebase credential must use the iOS client ID, not the web client ID.
Fix the audience mismatch. Ensure:
- Nonce is generated before the Apple auth request
- SHA256 hash of nonce is passed to Apple
- Raw nonce is passed to Firebase OAuthProvider credential
- Firebase signInWithCredential receives the correct credential with matching audience
- On iPad: presentation anchor must be set (window or specific UIViewController)
- On failure: show a visible error message — no silent failures

Do NOT break Google Sign-In or email/password auth.

==================================================
PHASE 3 — FIX UPGRADE TO SUPER HERO PURCHASE ERROR
==================================================

Product ID: MCV_Apple_Superhero

Requirements:
- Products must be fetched from Apple at app startup (or at latest when upgrade screen loads)
- The purchase button must be DISABLED or show a loading state until the product is confirmed available
- If product is unavailable after fetch: show a meaningful message, not a generic error
- If user cancels: handle gracefully, no error shown
- If purchase succeds: update user plan in database, unlock Super Hero features
- Do NOT break Stripe for web or Android — iOS must use Apple IAP only
- Do NOT show any Stripe UI on iOS
