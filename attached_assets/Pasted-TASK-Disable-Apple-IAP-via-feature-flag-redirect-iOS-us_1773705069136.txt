PHASE 4 — DEEP LINK RETURN FLOW
==================================================

When a user subscribes on the web, they land on the 
subscription success page. We want to automatically 
return them to the app and refresh their subscription status.

STEP A — Backend: update the Stripe success redirect URL
In server/routes.ts, find where the Stripe checkout session 
is created (POST /api/create-checkout-session).

Change the success_url to include a flag:
success_url: `${process.env.CLIENT_URL}/subscription-success?from_ios=true`

Or if CLIENT_URL isn't set, use:
success_url: `https://marvelcardvault.com/subscription-success?from_ios=true`

STEP B — Frontend: subscription-success page
In client/src/pages/subscription-success.tsx (or wherever 
the success page lives), add this logic on mount:

import { Capacitor } from '@capacitor/core';
import { App } from '@capacitor/app';

useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  const fromIos = params.get('from_ios');
  
  if (fromIos === 'true') {
    // Short delay so Stripe webhook has time to process
    setTimeout(() => {
      // Try deep link back to app
      window.location.href = 'marvelcardvault://subscription-success';
      
      // Fallback: if deep link doesn't fire within 2s, show manual prompt
      setTimeout(() => {
        // Show "Tap here to return to the app" button
        setShowReturnButton(true);
      }, 2000);
    }, 1500);
  }
}, []);

Add state: const [showReturnButton, setShowReturnButton] = useState(false);

Add UI below the success message:
{showReturnButton && (
  <button onClick={() => window.location.href = 'marvelcardvault://subscription-success'}>
    Tap here to return to Marvel Card Vault
  </button>
)}

STEP C — Capacitor deep link handler (App.tsx)
Import App from @capacitor/app (already likely installed).

Add this useEffect to the App() component:

useEffect(() => {
  if (!Capacitor.isNativePlatform()) return;
  
  const handleDeepLink = App.addListener('appUrlOpen', (data) => {
    if (data.url.includes('subscription-success')) {
      // Invalidate subscription queries to force a refresh
      queryClient.invalidateQueries({ queryKey: ['/api/auth/me'] });
      queryClient.invalidateQueries({ queryKey: ['/api/subscription/status'] });
      // Navigate to dashboard
      window.location.hash = '/';
    }
  });
  
  return () => {
    handleDeepLink.then(l => l.remove());
  };
}, []);

PHASE 5 — SUBSCRIPTION STATUS AUTO-REFRESH
==================================================

When the user returns to the app (via deep link OR manually), 
their subscription status needs to refresh.

In client/src/App.tsx, add this to the App() component:

import { App as CapApp } from '@capacitor/app';

useEffect(() => {
  if (!Capacitor.isNativePlatform()) return;
  
  // When app comes back to foreground, refresh subscription status
  const handleResume = CapApp.addListener('appStateChange', (state) => {
    if (state.isActive) {
      queryClient.invalidateQueries({ queryKey: ['/api/auth/me'] });
      queryClient.invalidateQueries({ queryKey: ['/api/subscription/status'] });
    }
  });
  
  return () => {
    handleResume.then(l => l.remove());
  };
}, []);

This means: even if the deep link doesn't fire perfectly, 
the moment the user comes back to the app from Safari, 
their subscription status refreshes automatically.
